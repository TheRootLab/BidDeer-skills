# I/O Contract: proposal-point-checker

## Input: 人工审核点清单 (Checklist)

| 审核项 ID | 审核要求 / 审核点描述 | 重点关注关键词 (可选) |
| :--- | :--- | :--- |
| T-01 | [描述要求] | [关键词1, 关键词2] |

## Output: AI 检查结果总表

| 审核项 ID | 证据状态 | 审核要求 | 标书体现 / 对齐结果 | 文件出处 | 核对提示 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| T-01 | [状态] | [要求] | [对齐描述] | [页码/章节] | [提示] |

## 证据状态枚举

* **符合 (Compliant)**: 找到明确证据且完全对齐要求。
* **部分符合 (Partial)**: 找到证据 but 存在细微偏差。
* **不符合 (Non-Compliant)**: 找到证据 but 明确违反要求。
* **未提及 (Not Found)**: 遍历全表未找到相关表述。
* **需人工确认 (Manual Check)**: 找到相关表述 but 无法自动判定对齐度。

## 安全与边界

1. **隐私保护**: 严禁在输出中包含真实项目敏感信息（如金额、人员联系方式）。
2. **辅助定位**: 输出结果仅作为人工核对的线索定位，不作为最终合规性结论。
